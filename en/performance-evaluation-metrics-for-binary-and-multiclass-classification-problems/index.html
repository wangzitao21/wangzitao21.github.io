Binary Classification#

Why F1 Score instead of Accuracy?

Consider a disease screening scenario where the prevalence of a certain disease is 5%, i.e., 5 out of 100 people have the disease, and 95 are healthy.

A model is needed to help doctors in diagnosis. The model is simple; it always predicts ‘healthy’ for all patients and never predicts ‘diseased’. In other words, the model classifies everyone as ‘healthy’.

In this case, the model’s Accuracy would be as high as 95%, but it is not a good model.

The above illustrates why F1 Score is introduced in classification problems.

Binary classification can be evaluated using ROC-AUC.

Assume a binary classification problem with labels Positive and Negative. The confusion matrix can be written as:

PositiveNegative
PositiveTrue Positive (TP)False Positive (FP)
NegativeFalse Negative (FN)True Negative (TN)

In the table above, columns represent predicted outcomes, and rows represent actual outcomes. FN and FP are cases of incorrect predictions.

Accuracy is the ratio of correct predictions to all predictions:

$$ Accuracy = \frac{TP + TN}{TP + TN + FP + FN} $$

Precision is the ratio of correctly predicted Positive samples to all samples predicted as Positive:

$$ Presion = \frac{TP}{TP+FP} $$

Recall is the ratio of correctly predicted Positive samples among all actual positive samples. Hence, Recall is also known as the True Positive Rate (TPR).

$$ Recall = TPR = \frac{TP}{TP + FN} $$

The recall rate for Negative samples is the True Negative Rate (TNR):

$$ TNR = \frac{TN}{TN + FP} $$

Incorrectly identifying positives is the False Positive Rate (FPR). The common ROC curve is plotted with FPR on the x-axis and TPR on the y-axis.

$$ FPR = \frac{FP}{TN + FP} = 1 - TNR $$

Specificity is similar to Recall, and it’s the proportion of actual Negative samples that are correctly predicted as Negative:

$$ Specificity = \frac{TN}{TN + FP} $$

F1 Score is a metric that combines Precision and Recall and provides a more comprehensive evaluation:

$$ F1 Score = \frac{2\times Precision\times Recall}{Precision + Recall} $$

In the disease example mentioned above, the F1 Score would be 0.

Multiclass Classification#

In multiclass classification problems, the confusion matrix also applies.

Taking animals as an example, suppose the classification task is for a machine to classify three types of animals: pigs, dogs, and cats. If the confusion matrix is as follows:

CatDogPig
Cat1012
Dog3154
Pig5620

Similar to binary classification, the horizontal direction represents predicted results, and the vertical direction represents actual results.

The Accuracy is calculated as:

$$ Accuracy = \frac{10 + 15 + 20}{10 + 15 + 20 + 1 + 2 + 3 + 4 + 5 + 6} = 68.2% $$

Additionally, in multiclass classification, Precision, Recall, and Specificity need to be calculated individually for each class. For example, using only cats to evaluate the confusion matrix:

CatNot Cat
Cat103
Not Cat845

Precision, Recall, Specificity, and F1 Score specific to cat classification are:

  • Precision is $10/(10+3)=76.9%$.
  • Recall is $10/(10+8)=55.6%$.
  • Specificity is $45/(45+3)=93.8%$.
  • F1 Score is $(2\times 0.769\times 0.556)/(0.769 + 0.556)=64.54%$.

The same method can be used to calculate the metric values for pigs and dogs.

Based on the above method, when using multiclass classification for evaluation, we can calculate the metric value for each class and then average these values.

Taking Precision as an example, the macro-average Precision is:

$$ Precison_{\text {macro}}=\frac{1}{K} \sum_{i=1}^K Precison_i=\frac{1}{K} \sum_{i=1}^K \frac{T P_i}{T P_i+F P_i} \ $$

Macro-averaging involves treating the evaluation of $n$ categories as $n$ evaluations of binary classifications, computing the Precision for each binary classification, and then averaging the $n$ Precision values to get the macro-average Precision.

Micro-average Precision is:

$$ Precision_{\text {micro }}=\frac{\sum_\limits{i=1}^K T P_i}{\sum_\limits{i=1}^K\left(T P_i+F P_i\right)} $$

Micro-averaging involves breaking down the $n$-class evaluation into $n$ binary evaluations, adding up the TP and (TP+FP) values of each of the $n$ binary classification evaluations respectively, and then dividing.

The effects of Macro and Micro averages are different.

Micro considers the quantity of each category in the calculation formula, so it is suitable for situations where the data distribution is imbalanced; however, due to the consideration of the quantity of data, in cases where the data is extremely imbalanced, the categories with a large number of quantities will have a greater impact on the evaluation results.

Macro does not consider the quantity of the data, so it treats each class equally (because the Precision and Recall of each class are between 0 and 1), and is relatively more influenced by classes with high Precision and high Recall.

In Sklearn, Recall, Precision, and F1 Scores all support the aforementioned metric calculations.

from sklearn.datasets import load_iris
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import train_test_split
from sklearn.metrics import precision_score, recall_score, f1_score

iris = load_iris()
X_train, X_test, y_train, y_test = train_test_split(iris.data, iris.target, test_size=0.3, random_state=42)

clf = DecisionTreeClassifier(random_state=0)
clf.fit(X_train, y_train)
y_pred = clf.predict(X_test)

micro_precision = precision_score(y_test, y_pred, average='micro')
macro_precision = precision_score(y_test, y_pred, average='macro')
print("Micro precision score:", micro_precision)
print("Macro precision score:", macro_precision)

micro_recall = recall_score(y_test, y_pred, average='micro')
macro_recall = recall_score(y_test, y_pred, average='macro')
print("Micro recall score:", micro_recall)
print("Macro recall score:", macro_recall)

micro_f1 = f1_score(y_test, y_pred, average='micro')
macro_f1 = f1_score(y_test, y_pred, average='macro')
print("Micro F1 score:", micro_f1)
print("Macro F1 score:", macro_f1)

For ROC curves in multiclass classification, detailed information is available in the Sklearn documentation: Multiclass Receiver Operating Characteristic (ROC). Here is a paraphrase of that document:

In binary classification, FPR is on the x-axis, and TPR is on the y-axis. This means that the top-left corner of the graph is the best point (where FPR is 0 and TPR is 1). Therefore, the larger the AUC value under the image, the better.

In multiclass classification, the ROC is theoretically not directly applicable. However, the problem can be solved by binarizing the multiclass problem. There are two approaches:

  1. Compare each class with all other classes, i.e., One-vs-Rest.
  2. Compare each pair of classes individually, i.e., One-vs-One.

We will demonstrate this using an iris flower dataset. Here, we will binarize the output and add noise features to increase the complexity of the problem.

import numpy as np
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split

iris = load_iris()
target_names = iris.target_names
X, y = iris.data, iris.target
y = iris.target_names[y]

# Added noise randomly
random_state = np.random.RandomState(0)
n_samples, n_features = X.shape
n_classes = len(np.unique(y))
X = np.concatenate([X, random_state.randn(n_samples, 200 * n_features)], axis=1)
(
    X_train,
    X_test,
    y_train,
    y_test,
) = train_test_split(X, y, test_size=0.5, stratify=y, random_state=0)

The other two classes can also be plotted in the same manner.

The above method performs plotting for a single class; in addition, Micro and Macro averaging can be used to plot as well.

For the One-vs-One scenario, a classifier needs to be built for each class pair. A total of $n\times (n - 1) / 2$ classifiers need to be trained and are therefore slower than One-vs-Rest.

In the iris dataset, the One-vs-One scheme is demonstrated using the 3 possible combinations from the iris plant dataset: setosa vs versicolor, versicolor vs virginica, and virginica vs setosa, specifically:
