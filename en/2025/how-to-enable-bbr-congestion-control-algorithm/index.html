How to Enable BBR Congestion Control Algorithm

Article description, related to search engine optimization
2025-01-11 · Zitao Wang | Translations:
  • 🌐 切换为中文

BBR (Bottleneck Bandwidth and Round-trip propagation time) is a TCP congestion control algorithm proposed by Google in 2016, which optimizes network performance by estimating bandwidth and round-trip delay.

Traditional congestion control algorithms such as Reno and Cubic rely on packet loss to determine network congestion. However, packet loss is not only caused by network congestion, and it is less efficient in high-bandwidth, high-latency networks.

For details on how it works, see bbr Congestion Control Optimizes Link Speed.

Here are the steps to enable BBR:

  1. Check the current TCP congestion control algorithm:
0
1

sysctl net.ipv4.tcp_congestion_control
# The default is net.ipv4.tcp_congestion_control = cubic
  1. Enter the following commands to modify the /etc/sysctl.conf file and permanently enable BBR:
0
1

echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf
sysctl -p
  1. BBR is more compatible with queue schedulers fq or fq_codel, so set the queue scheduler to FQ (Fair Queueing):
0
1

echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf
sysctl -p
  1. Check the current TCP congestion control algorithm again:
0
1

sysctl net.ipv4.tcp_congestion_control
# net.ipv4.tcp_congestion_control = bbr
