Hugo 安装#

Windows 版本的 hugo 安装比较简单，直接在官网下载 hugo_extended 后，将其解压，然后配置环境变量即可。

Linux (Debian) 系统的 Hugo 仓库版本有点老，故采用以下命令安装:

# 链接需要更改
wget https://github.com/gohugoio/hugo/releases/download/v0.140.2/hugo_0.140.2_linux-amd64.deb

# 解压
sudo dpkg -i hugo_0.140.2_linux-amd64.deb

无论是 Windows 还是 Linux，配置完成均可采用以下命令检查是否安装成功:

hugo version

本地配置#

初次部署步骤如下:

  1. 先在 Git 命令里操作，首先移动到需要建博客的指定位置然后新建站点。
cd /e/onedrive

# blog 名可以更改
hugo new site blog --format yaml
  1. 进入主题文件夹下载安装主题。该主题具体配置见 PaperMod。
cd /e/onedrive/blog
git clone https://github.com/adityatelange/hugo-PaperMod themes/PaperMod --depth=1

在 hugo.yaml 配置文件中添加主题名:

baseURL: https://example.org/
languageCode: en-us
title: My New Hugo Site
theme: ["PaperMod"]
  1. 此时已经安装完成了，可以测试一下:
cd /e/onedrive/blog

# 如果是在服务器上可以修改 IP
hugo server --bind=0.0.0.0 --baseURL=http://127.0.0.1 --port=1313

非初次部署是指已经有 hugo 博客文件夹了。部署步骤基本一致，从上述第 2 步开始即可。

Github 部署#

  1. 首先需要先建一个 Github 仓库。
  2. 切换到 pubilc 文件夹。
cd /e/onedrive/blog/public
  1. 首次部署需要初始化仓库。
# 注意初始化是在 public 目录不是 blog 目录
git init
# 输出为 Initialized empty Git repository in E:/OneDrive/blog/public/.git/
  1. 连接仓库。
# 先测试一下，这时候应该输出空白
git remote -v

# 连接远程仓库
git remote add origin git@github.com:wangzitao21/wangzitao21.github.io.git

# 再测试一下
git remote -v
# 此时输出如下:
# origin  git@github.com:wangzitao21/wangzitao21.github.io.git (fetch)
# origin  git@github.com:wangzitao21/wangzitao21.github.io.git (push)
  1. 测试一下本地渲染效果:
# 先切回目录
cd /e/onedrive/blog
hugo server --bind=0.0.0.0 --baseURL=http://192.168.6.66 --port=1313
  1. 每次部署按以下步骤部署即可:
# 先在 blog 目录部署
cd /e/onedrive/blog
hugo -D

# 切到 public 目录上传
cd /e/onedrive/blog/public
# 没连上用这个
git remote add origin git@github.com:wangzitao21/wangzitao21.github.io.git

git add .
git commit -m "update"
# git branch -M main
git push -u origin main
git push -u origin main --force

# 直接一句搞定
cd /e/onedrive/blog && hugo -D && cd /e/onedrive/blog/public && git add . && git commit -m "update" && git push -u origin main --force

添加搜索页面#

参考 官方文档 介绍。具体步骤如下:

  1. 在 config.yml 文件中添加以下内容:
outputs:
    home:
        - HTML
        - RSS
        - JSON # is necessary
  1. 在 content 文件夹中创建 search.md 并添加以下内容:
---
title: "Search" # in any language you want
layout: "search" # necessary for search
# url: "/archive"
# description: "Description for Search"
summary: "search"
placeholder: "placeholder text in search input box"
---

或:

---
title: "🔍 搜索" # in any language you want
layout: "search" # is necessary
# url: "/archive"
# description: "Description for Search"
summary: "search"
placeholder: "搜索框内的默认显示"
---

若某个特定文章不被搜索到，需在文章开头添加以下配置:

searchHidden: true

自定义模板#

Hugo 提供了一个默认的文章使用模板，位于 archetypes/default.md，里面包括一些基本内容标题、日期以及是否为草稿可以修改。

我们也可以自定义其他的模板，在生成新文章的时候，如果要指定模板 new，可以输入:

hugo new --kind post content/Chinese/post/XXX.md

hugo new --kind post_zh content/Chinese/XXX.md
hugo new --kind post_en content/English/XXX.md

Cloudflare CDN 加速#

Github Page 国内直接访问速度很慢，有时候更是彻底被封了。而如果使用国内 CDN 的话，一方面需要付费，另一方面则需要将域名备案。

所以折中的方法就是用 Cloudflare 的免费 CDN。

Cloudflare 的配置非常简单，只需要在域名 > DNS 处添加对 Github Page 的解析。即:

Type Name content
A blog 185.199.108.153
A blog 185.199.109.153
A blog 185.199.110.153
A blog 185.199.111.153

然后将各项后面的小黄云开启即可。

在这过程中我遇到一个问题，明明配置正常，但访问博客时会出现，会提示 Invalid SSL certificate (526) 报错。经查，是因为在域名 > SSL/TLS > 配置处开启了 完全 (严格) 项。改成 完全 后方才正常。

经 Cloudflare CDN 加速后，延迟虽然还有近 200ms，但贵在稳定，访问完全没问题了。

可以使用这个 站长工具 测试一下各地访问的延迟。
