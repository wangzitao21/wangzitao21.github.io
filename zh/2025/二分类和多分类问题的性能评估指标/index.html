二分类#

为什么需要 F1 Score，而不是 Accuracy?

以患病筛查为例，若某种疾病患者比例为 5%，即每 100 人中有 5 个人患病，95 个人健康。

现需建立一模型帮助医生去进行诊断。该模型很简单，它只会将求诊者预测为健康，而不会预测为患病。换句话说，该模型将所有人全部分类为健康。

此时该模型 Accuracy 能高达 95%，却并非属于好模型。

以上说明了为什么在分类问题上，要引入 F1 Score 的原因。

二分类可以使用 ROC-AUC 来评价其效果。

假设某二分类问题标签为 Positive 和 Negative，则混淆矩阵可写为:

Positive Negative
Positive True Positive (TP) False Positive (FP)
Negative False Negative (FN) True Negative (TN)

上表中，列名为预测情况，行名为真实情况。其中 FN 和 FP 属预测错误的情况。

则 Accuracy 为预测正确的情况占所有情况的比值:

$$ Accuracy = \frac{TP + TN}{TP + TN + FP + FN} $$

Precision 表示预测正确的 Positive 样品占所有被预测为 Positive 样品的比例:

$$ Presion = \frac{TP}{TP+FP} $$

Recall 是实际为 Positive 的样品中，被正确预测的比例。因而 Recall 又被称为真阳性率 (True Positive Rate, TPR)。

$$ Recall = TPR = \frac{TP}{TP + FN} $$

如果是 Negative 样品的召回率，那就是真阴性率 (True Negative Rate, TNR):

$$ TNR = \frac{TN}{TN + FP} $$

而错误的识别 Positive 就是假阳性率 (False Positive Rate, FPR)。常用的 ROC 曲线是以 FPR 为横轴，TPR 为纵轴所绘制。

$$ FPR = \frac{FP}{TN + FP} = 1 - TNR $$

Specificity 特异性和 Recall 相对，是实际为 Negative 的样品中，被正确预测的比例:

$$ Specificity = \frac{TN}{TN + FP} $$

F1 Score 是将 Precision 和 Recall 整合在一起的判定标准，相对更加科学：

$$ F1 Score = \frac{2\times Precision\times Recall}{Precision + Recall} $$

在上述患病的例子中，F1 Score 会判定为 0。

多分类#

在多分类问题中，混淆矩阵也是成立的。

以动物为例，假如分类问题是让机器分类猪、狗和猫三种动物。若混淆矩阵为:

Cat Dog Pig
Cat 10 1 2
Dog 3 15 4
Pig 5 6 20

和二分类一样，横向为预测结果，纵向为实际情况。

则 Accuracy 为：

$$ Accuracy = \frac{10 + 15 + 20}{10 + 15 + 20 + 1 + 2 + 3 + 4 + 5 + 6} = 68.2% $$

此外，多分类中的 Precision，Recall 和 Specificity 需要针对每一类单独计算。以猫为例，单独用猫来评估的混淆矩阵为：

Cat Not Cat
Cat 10 3
Not Cat 8 45

单独针对猫分类的 Precision，Recall, Specificity 和 F1 Score 可被计算为：

  • Precision 为 $10/(10+3)=76.9%$。
  • Recall 为 $10/(10+8)=55.6%$。
  • Sprcificity 为 $45/(45+3)=93.8%$。
  • F1 Score 为 $(2\times 0.769\times 0.556)/(0.769 + 0.556)=64.54%$。

同样的方法可以计算猪与狗各自的指标值。

基于上述方案，使用多分类来评估时，可分别求每类指标值后求平均。

以 Precision 指标为例，如 Precision 的宏平均:

$$ Precison_{\text {macro}}=\frac{1}{K} \sum_{i=1}^K Precison_i=\frac{1}{K} \sum_{i=1}^K \frac{T P_i}{T P_i+F P_i} \ $$

宏平均就是将 $n$ 个分类的评价拆成 $n$ 个二分类的评价，计算每个二分类的 Precision，$n$ 个 Precision 平均值即为 Precision 的宏平均。

Precision 的微平均:

$$ Precision_{\text {micro }}=\frac{\sum_\limits{i=1}^K T P_i}{\sum_\limits{i=1}^K\left(T P_i+F P_i\right)} $$

微平均是将 $n$ 分类的评价拆成 $n$ 个二分类的评价，将 $n$ 个二分类评价的 TP 和 (TP+FP) 对应相加，然后再相除。

Macro 和 Micro 的效果不同。

Micro 在计算公式中考虑到了每个类别的数量，所以适用于数据分布不平衡的情况；但同时因为考虑到数据的数量，所以在数据极度不平衡的情况下，数量较多数量的类会较大的影响到评估结果。

Marco 没有考虑到数据的数量，所以会平等的看待每一类（因为每一类的 Precision 和 Recall 都在 0-1 之间），会相对受高 Precision 和高 Recall 类的影响较大。

from sklearn.datasets import load_iris
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import train_test_split
from sklearn.metrics import precision_score, recall_score, f1_score

iris = load_iris()
X_train, X_test, y_train, y_test = train_test_split(iris.data, iris.target, test_size=0.3, random_state=42)

clf = DecisionTreeClassifier(random_state=0)
clf.fit(X_train, y_train)
y_pred = clf.predict(X_test)

micro_precision = precision_score(y_test, y_pred, average='micro')
macro_precision = precision_score(y_test, y_pred, average='macro')
print("Micro precision score:", micro_precision)
print("Macro precision score:", macro_precision)

micro_recall = recall_score(y_test, y_pred, average='micro')
macro_recall = recall_score(y_test, y_pred, average='macro')
print("Micro recall score:", micro_recall)
print("Macro recall score:", macro_recall)

micro_f1 = f1_score(y_test, y_pred, average='micro')
macro_f1 = f1_score(y_test, y_pred, average='macro')
print("Micro F1 score:", micro_f1)
print("Macro F1 score:", macro_f1)

关于多分类的 ROC 曲线，Sklearn 文档中有详细介绍: Multiclass Receiver Operating Characteristic (ROC)。以下是对该文档的意译:

在二分类问题时，FPR 在 $x$ 轴，TPR 在 $y$ 轴。这意味着图像的左上角是最佳点 （此时 FPR 为 0，TPR 为 1）。从而图像下的 AUC 值越大越好。

在多分类问题中，ROC 理论上已不可用。但是可以将多分类问题进行二进制化的方式来解决，有两种方案:

  1. 将每类和所有其他类比较，即 One-vs-Rest。
  2. 将每两类单独比较，即 One-vs-One。

下面通过一个鸢尾花数据集来演示。在这里，对输出进行二进制化处理，并添加噪声特征以增加问题的难度。

import numpy as np
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split

iris = load_iris()
target_names = iris.target_names
X, y = iris.data, iris.target
y = iris.target_names[y]

# 随机增加了噪声
random_state = np.random.RandomState(0)
n_samples, n_features = X.shape
n_classes = len(np.unique(y))
X = np.concatenate([X, random_state.randn(n_samples, 200 * n_features)], axis=1)
(
    X_train,
    X_test,
    y_train,
    y_test,
) = train_test_split(X, y, test_size=0.5, stratify=y, random_state=0)

另外两种类型也可以以相同形式绘制。

上述的做法是对单独一类的绘制，此外也可以 Micro 和 Macro 的方式求平均绘制。

对于 One-vs-One 类，需要为每个类对安装一个分类器。共需训练 $n\times (n - 1) / 2$ 个分类器，故比 One-vs-Rest 更慢。

在鸢尾花数据集中，使用 One-vs-One 方案演示了鸢尾植物数据集中 3 种可能组合：setosa 与 versicolor、versicolor 与 virginica 以及 virginica 与 setosa，即:
