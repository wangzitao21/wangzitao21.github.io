服务器和域名

本次是在 NameSilo 上购买的域名。

域名购买非常简单，购买完毕后，依次选择 Manage My Domains > Manage DNS for this domain 即可配置 DNS。

服务器可在 AWS 上申请三个月免费，可利用 美国身份随机生成器。

如果有 Visa 卡，也可以在 Google Cloud 上申请免费一年的服务器。

AWS 服务器和国内服务器有个较大的不同，即当服务器新建后，AWS 只能以 user 名义进入服务器，即 ubuntu@<ip address>。

然后才可以通过 sudo passwd root 来创建密码，并用 su root 切换到 root 账户。

服务器配置

Windows Terminal

在云服务器平台生成密钥对，将云服务器平台将密钥对和服务器绑定，并 Download default key。

然后将密钥 .pem 文件需放在 C:\\Users\\wang 或 C:\\Users\\wang\\.ssh 中，否则可能会失效。

此外，为了确保 Windows 可以使用 ssh 命令，需把 Openssh 添加至环境变量，即将 C:\Windows\System32\OpenSSH 添加到 System variables > Path 中。

然后打开 Windows Terminal 的 setting.json，在 "list" 添加:

  • commandline 中，为 ssh -i <.pem 绝对路径> <用户名>@<ip address>。
  • guid 可在 在线生成GUID 中生成。
  • icon 和 name 随意。
  • 其他可参考 Dynamic profiles in Windows Terminal。

PuTTY

安装 PuTTY，这是一款开源免费的 SSH 客户端。

安装完毕后配置以下内容:

  • 打开 PuTTYgen > Load > 导入 .pem > Save private key > 保存为 .ppk 格式密钥。
  • 打开 PuTTY > PuTTY Configuration:
    • Session > Host Name (or IP address): 云服务器的地址 > Port: 22 > Connection type: SSH
    • Connection > Data > Auto-login username: root 或 用户名
    • Connection > SSH > Auth > Browse: .ppk
    • Session > Saved Sessions: 配置名 > Save

完成上述步骤后，点击 Open 即可连接上 PuTTY。

除了基本的配置外，PuTTY 初始界面比较丑，可对其进行美化:

  • Window > Font settings > Font: Consolas, 14-point
  • Window > Behaviour > Checking System appears on ALT-Space
  • Window > Selection > Assign copy/paste actions to clipboard:
    • Mouse paste action: Last selected text
    • {Ctrl, Shift} + Ins: No action
    • Ctrl + Shift + {C, V}: System clipboard
  • Window > Colours > Select a colour to adjust:
    • Default Foreground: 255, 255, 255
    • ANSI Blue: 0, 128, 255
    • ANSI Blue Bold: 0, 128, 255
    • ANSI Magenta: 255, 128, 255
    • ANSI Magenta Blod: 255, 128, 255

PuTTY 的配置可以导出，并加载到其他电脑里:

  • Win + R 并输入 regedit，打开 Registry Editor
  • 找到 Computer\HKEY_CURRENT_USER\SOFTWARE\SimonTatham\PuTTY
  • 右键 PuTTY > Export > 保存为 PuTTY.reg
  • 在其他电脑，右键 PuTTY.reg > Merge 即可

搭建邮箱服务器

网上查到的构建邮箱服务器方法有多种。

一种简单的方法、

该简单的方法即直接用 Hestia。

  • 打开 Hestia > 进入新建的用户账户
  • MAIL > Add Mail Domain > 输入 Domain，Webmail Cilent 默认选择为 Roundcube Webmail

假如我们要建一个 @mail.mydomain.tld 后缀的邮箱，则在 Domain 项目直接输入 mydomain.tld 即可。
Hestia 会自动为我们建立两个子域名，一个是 webmail.mymain.tld 的网站，一个是 @mail.mydomain.tld 的邮箱后缀。

  • 打开该邮箱的 DNS records，会发现 Hestia 已经写好了邮箱的 DNS 配置方案，按照其方式前往域名购买官网去配置 DNS，即:
record type IP or value
mail.mydomin.tld A ip address
webmail.mydomain.tld A ip address
mydomain.tld MX mail.mydomin.tld
  • 点击 Add Mail Account 来创建邮箱。
  • 前往安全组开放端口，包括 25、109、110、143、465、587、993、995 等。
  • 完成上述步骤后打开 webmail.mydomain.tld 即可访问邮箱网站并登录。

邮箱搭建完成后，我遇到一个问题，即邮箱只能接收其他邮箱 (163, gmail, edu 等) 发来的邮件，却不能成功发送邮件。

但是我自己建的两个账号之间是可以互发邮件的。

如果我发送到其他邮箱后，Roundcube Webmail 仍旧会提示发送成功，但其他邮箱收不到，也不存在于垃圾邮件的分类里。

网上搜索该问题后，有人提出将 SMTP Relay 开启，并将其 Host 改成 127.0.0.1，Port 改成 465 或 587 或 25，均不奏效。

该问题待解决。

补充: 在 Hestia 论坛咨询后得知，可能是需要开通 Amazon SES 服务，才可正常发送邮件。

搭建代理服务器

一种简单的方法

在上述 PuTTY 配置过程中，添加以下内容:

  • Connection > SSH > Tunnels > Checking Auto, Dynamic > Source port: 77777 或其他不冲突的端口号均可 > Add
  • Session > Saved Sessions: 配置名 > Save

然后打开浏览器的 Proxy SwitchyOmega 插件，在选项中配置以下内容:

  • 新建情景模式 > 输入情景模式名称: new_proxy
  • 选择情景模式的类型: 代理服务器 > 创建
  • 代理协议: SOCKS5 > 代理服务器: 127.0.0.1 > 代理端口: 77777
  • 应用选项

开启 new_proxy，打开 what is my ip address 查询，此时已是代理服务器的地址。

该方法来自 代理服务器搭建， Use PuTTY as a secure proxy on Windows 以及 Create your own Virtual Private Network for SSH with Putty。
